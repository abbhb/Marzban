# 用户门户、钱包与套餐设计

本文定义 Marzban MGMA fork 的自助用户门户。实现必须遵循这里的数据、权限和事务契约，不能把门户用户当成面板管理员，也不能把钱包余额当作可直接覆盖的普通字段。

## 1. 范围与角色

- 门户用户可以注册、登录，查看余额、钱包流水、当前套餐、可购买套餐、本人流量，并获取或撤销本人的 MGMA 临时订阅链接。
- sudo 管理员可以维护套餐、给门户用户分配唯一可见套餐、充值、覆盖开通套餐和续期。
- 普通 Marzban 管理员不获得钱包或门户管理权限。
- 本版本不接入外部支付网关；充值只能由 sudo 管理员完成。

## 2. 核心规则

1. 一个门户账户最多关联一个 Marzban 代理用户，并最多被分配一个可见套餐。
2. 注册只创建门户账户。首次开通套餐时才创建同名 VLESS 用户，避免未付费账户进入 Xray 配置。
3. 用户套餐列表只能返回管理员分配给该用户且已上架的套餐，结果只能是零或一条。购买接口仍会再次校验分配关系和上架状态。
4. 自助购买按“覆盖”处理：余额扣除成功后，以购买时刻重新计算到期时间，流量归零，覆盖流量上限与 inbound 权限；已有 VLESS UUID 保持不变。
5. 管理员续期不扣钱包，只把当前到期时间向后延长指定天数；已过期套餐从当前时间开始延长。
6. 金额以最小货币单位存储，当前货币固定为 `CNY`，例如 `1299` 表示 `¥12.99`。禁止用浮点数保存金额。
7. 每次余额变化必须写入钱包流水。充值金额为正数，购买金额为负数；流水保存变更后余额和操作人，不能编辑或删除。
8. 自助购买、管理员充值、覆盖开通和续期都要求 `Idempotency-Key`。同一账户重复提交同一个键只返回第一次结果，不再次扣款、充值或延长；复用键但更换关键参数会返回冲突。

## 3. 数据模型

### `portal_accounts`

- 登录名、密码摘要、启用状态；
- 当前钱包余额（分）；
- 唯一可见套餐 `assigned_plan_id`；
- 一对一代理用户 `user_id`；
- `password_reset_at` 用于使旧 token 失效。

### `subscription_plans`

- 名称、说明、价格、货币、有效天数、流量上限；
- VLESS inbound tag 列表；
- 上架状态；
- 已有购买记录引用的套餐只下架，不硬删除。

### `portal_subscriptions`

- 每个账户至多一条当前套餐；
- 保存计划快照、购买时间、当前开始/到期时间和最近一次购买；
- 自助购买与管理员覆盖开通都会覆盖该记录，管理员续期只更新到期时间。

### `portal_purchases`

- 不可变购买历史；
- 保存幂等键、价格、余额前后值和购买时套餐快照。

### `wallet_transactions`

- 不可变余额流水；
- 保存有符号金额、变更后余额、类型、管理员操作人、购买引用、备注和时间。

## 4. API 边界

门户接口使用独立的 `portal` JWT，不能调用 `/api/admin/*`。除注册和登录外，所有门户接口都从 token 获取当前账户，不接收可替换的用户名。

- `POST /api/portal/register`
- `POST /api/portal/token`
- `GET /api/portal/me`
- `GET /api/portal/plans`
- `GET /api/portal/wallet/transactions`
- `POST /api/portal/purchase`
- `POST /api/portal/mgma`、`DELETE /api/portal/mgma`

sudo 管理接口统一位于 `/api/commerce/admin/*`：

- 套餐列表、新建和修改；
- 门户账户列表；
- 分配唯一可见套餐；
- 充值；
- 覆盖开通、续期和停用当前套餐。

## 5. 事务与安全

- 购买用数据库条件更新完成原子扣款：只有 `balance >= price` 时余额才会变化。
- 扣款、钱包流水、购买历史、当前套餐和代理用户覆盖在同一数据库事务内提交；任一步失败均回滚。
- Xray 更新只在数据库提交后执行。失败时数据库仍是事实来源，管理员可以重试同步，不重复扣款。
- 门户密码使用与管理员相同的 bcrypt 实现，注册密码至少 10 个字符；响应和日志不得包含密码摘要、JWT、MGMA token 或 VLESS UUID。
- 注册按真实客户端 IP 限制为每分钟 5 次；登录按 IP 限制为每 5 分钟 60 次，并按 IP + 用户名限制为每 5 分钟 5 次，登录成功后清除该账号失败窗口。限流器最多保留 10000 个键，防止攻击者用随机用户名耗尽内存。
- MGMA 明文 token 仍只在签发响应中出现一次，数据库只保存 keyed digest；门户只可为自己的代理用户签发。

## 6. UI

- `#/portal/register`、`#/portal/login` 和 `#/portal` 构成用户门户。
- 门户首页展示余额、当前套餐、到期时间、已用/总流量、唯一可见套餐、钱包流水和 MGMA 获取按钮。
- 管理端新增“用户与套餐”页面，提供套餐维护、账户分配、充值、覆盖开通和续期。
- 所有新增文案必须同时加入 `en/fa/ru/zh` locale，构建测试会校验 key 集合一致，避免出现原始国际化 key。
